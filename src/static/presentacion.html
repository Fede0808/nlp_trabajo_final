01 · Problema y corpus

Muestreo y preprocesamiento

El proyecto aborda la clasificacion de avisos inmobiliarios a partir de texto libre (`description`) y busca predecir `Departamento`, `Casa` o `PH` con una configuracion reproducible.

Dataset operativo

`data/entrenamiento.csv` con texto y etiqueta objetivo.

Particion unificada

Muestra de 10.000 casos, con 8.000 en train y 2.000 en test.

Reproducibilidad

Semilla 42 y la misma particion para SVM y DistilBERT.

  • Preprocesamiento sobre la columna canonica `texto_limpio`.
  • Escenario censurado como control de data leakage.
  • Mismos casos para todos los modelos; cambia solo el tratamiento del texto.
02 · Baseline clasico

TF-IDF + SVM como artefacto principal

El baseline clasico es el modelo operativo del proyecto. Combina representacion TF-IDF con SVM y se persiste como `artifacts/modelo_svm.joblib` para inferencia local.

Accuracy Base 0.8992
F1 Macro Base 0.7904
Accuracy Censurado 0.8427
F1 Macro Censurado 0.7185
03 · Enfoque profundo

DistilBERT CPU-only bajo presupuesto controlado

El enfoque profundo se implemento con `distilbert-base-multilingual-cased`, ejecutado con `max_length=128`, `batch_size=4` y `epochs=1`.

Accuracy Base 0.8897
F1 Macro Base 0.7847
Accuracy Censurado 0.7254
F1 Macro Censurado 0.6265

El `LOAD REPORT` es esperable en fine-tuning sobre checkpoint base: la cabeza de clasificacion se inicializa desde cero.

04 · Comparacion metodologica

Resultados y leakage

Ambos enfoques se evaluaron sobre el mismo corpus, las mismas clases y el mismo split. El escenario censurado se uso para medir impacto del leakage sobre la señal linguistica.

Modelo Base Censurado
SVM 0.8992 / 0.7904 0.8427 / 0.7185
DistilBERT 0.8897 / 0.7847 0.7254 / 0.6265

Conclusion tecnica: el baseline SVM resulta mas resiliente bajo el presupuesto de computo actual y queda como artefacto principal de inferencia.
